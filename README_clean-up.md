# クリーンアップスクリプト使用ガイド

## 概要

このスクリプトは、Microsoft EdgeとBob関連のデータを安全に削除するための自動化ツールです。

## 機能

### 削除対象

1. **Microsoft Edge**
   - 実行中のEdgeプロセスを終了
   - ユーザーデータ（キャッシュ、履歴、設定など）を削除

2. **Bob関連データ**
   - `%USERPROFILE%\.bob`
   - `%LOCALAPPDATA%\.bobide`
   - `%LOCALAPPDATA%\.continue`
   - `%APPDATA%\IBM Bob`

3. **一時ファイル**
   - システムの一時フォルダ内のファイル

### 追加機能

- **エラーハンドリング**: 各操作の成功/失敗を検出
- **管理者権限チェック**: 権限不足の場合に警告を表示
- **プロセス確認**: Edgeが実行中かどうかを確認してから終了
- **存在確認**: 削除対象が存在するかを確認してから削除

## 使用方法

### 基本的な実行方法

1. **通常実行**
   ```
   clean-up.bat をダブルクリック
   ```

2. **管理者として実行（推奨）**
   - `clean-up.bat` を右クリック
   - 「管理者として実行」を選択

### コマンドラインからの実行

```cmd
cd /d "スクリプトのあるディレクトリ"
clean-up.bat
```

## 実行結果の確認

### コンソール出力

スクリプト実行中、以下のような情報が表示されます：

```

  [情報] "%LOCALAPPDATA%\.bobide" は存在しません
```

## 注意事項

### ⚠️ 重要な警告

1. **データの完全削除**
   - このスクリプトは指定されたデータを**完全に削除**します
   - 削除されたデータは復元できません
   - 実行前に重要なデータをバックアップしてください

2. **Edgeの設定**
   - Edgeのすべての設定、ブックマーク、履歴が削除されます
   - 必要に応じて事前にエクスポートしてください

3. **実行中のプロセス**
   - Edgeが実行中の場合、強制終了されます
   - 保存されていないデータは失われます

### 推奨事項

- **管理者権限で実行**: より確実な削除のため
- **Edgeを閉じる**: 事前に手動で閉じることを推奨
- **バックアップ**: 重要なデータは事前にバックアップ
- **ログ確認**: 実行後はログファイルで結果を確認

## トラブルシューティング

### よくある問題と解決方法

#### 1. 「アクセスが拒否されました」エラー

**原因**: 管理者権限が不足している

**解決方法**:
- スクリプトを右クリック → 「管理者として実行」

#### 2. Edgeプロセスが終了しない

**原因**: 複数のEdgeプロセスが実行中、または他のアプリケーションがEdgeを使用している

**解決方法**:
1. タスクマネージャーを開く（Ctrl+Shift+Esc）
2. すべてのEdgeプロセスを手動で終了
3. スクリプトを再実行

#### 3. 一部のファイルが削除できない

**原因**: ファイルが使用中、または権限不足

**解決方法**:
1. すべてのアプリケーションを閉じる
2. PCを再起動
3. 管理者権限でスクリプトを再実行

### 一時ファイルのクリーンアップを無効化

該当セクション（4. 一時ファイルのクリーンアップ）をコメントアウト：

```batch
REM del /F /Q "%TEMP%\*" >nul 2>&1
```


## 技術仕様

### 動作環境

- **OS**: Windows 10/11
- **必要な権限**: 管理者権限（推奨）
- **依存関係**: なし（標準のWindowsコマンドのみ使用）

### 使用コマンド

- `tasklist`: プロセスの確認
- `taskkill`: プロセスの終了
- `rmdir`: ディレクトリの削除
- `del`: ファイルの削除
- `net session`: 管理者権限の確認

